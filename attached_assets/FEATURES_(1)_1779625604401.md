## 1. Bot Telegram

### Perintah Pengguna
| Perintah | Fungsi |
|---|---|
| `/start` atau `/menu` | Membuka menu utama dengan banner toko |
| `/produk` atau `/products` | Melihat katalog produk |
| `/profile` | Melihat ID, saldo, role, dan tanggal bergabung |
| `/deposit` | Top-up saldo via payment gateway |
| `/orders` | Riwayat pembelian dan kode akun yang diterima |
| `/admin` | Akses cepat untuk administrator |

### Tampilan & Navigasi
- Setiap layar menampilkan banner toko (gambar) sebagai header
- Navigasi mulus antar menu menggunakan inline keyboard
- Edit caption di tempat (tanpa animasi hapus/kirim ulang)
- Format pesan menggunakan HTML dengan `<blockquote>` dan `<code>` untuk keterbacaan

### Katalog Produk
- Tampilan daftar produk format satu baris: `[1] Nama Produk - Stok: 9`
- Filter berdasarkan kategori dengan emoji
- Indikator stok: Tersedia / Terbatas / Habis
- Indikator diskon massal (contoh: "Beli 5+ hemat 10%")
- Tampilan varian produk dengan jumlah terjual per varian

### Alur Pembelian
1. Pilih produk dari katalog
2. Pilih varian (jika ada)
3. Tentukan jumlah (tombol +/-)
4. Pilih metode pembayaran: **Saldo** atau **QRIS**
5. Kode akun dikirim otomatis setelah pembayaran terverifikasi
6. Sistem rating bintang 1–5 setelah pembelian

### Kode Akun (Receipt)
- Format receipt seragam untuk semua metode pembayaran
- Label menggunakan `<code>` tag untuk keterbacaan
- Kode akun ditampilkan dalam blok `<pre>` (monospace) di luar blockquote
- Format: `Email: xxx | Password: yyy`

### Dukungan Grup
- Bot dapat berinteraksi di grup Telegram
- Tindakan sensitif (Profile, Deposit) diarahkan ke Private DM untuk keamanan
- Deep-link support (contoh: `t.me/revdstore_bot?start=deposit`)

## 3. Metode Pembayaran

### Saldo Internal (Wallet)
- Pengguna top-up saldo via gateway, lalu belanja pakai saldo
- Pembelian instan tanpa menunggu konfirmasi
- Transaksi dilindungi database transaction atomik (mencegah race condition)
- Deduct saldo dengan `WHERE balance >= totalPrice` (mencegah saldo minus)

### PakKasir (QRIS)
- Pembuatan QRIS otomatis via API PakKasir
- Konfirmasi pembayaran via webhook callback
- Kode akun dikirim otomatis setelah webhook diterima
- Kode dilepas kembali (`releaseReservedCodes`) jika transaksi gagal

### GoPay / QRIS Manual
- Pembuatan QR via `GOPAY_API_BASE`
- Pencocokan pembayaran berdasarkan `midtransToken`
- Kode dilepas otomatis jika order gagal tersimpan

### OrderKuota (QRIS + Unique Code)
- QRIS dengan kode unik (3 digit) via `QRIS.convertUnique()`
- Verifikasi via pengecekan mutasi rekening secara polling
- Kode dilepas otomatis jika order gagal tersimpan

### Deposit Manual (Admin Konfirmasi)
- User bisa request deposit manual
- Admin konfirmasi dari panel → saldo ditambahkan otomatis

## 4. Manajemen Produk & Stok

### Produk
- Buat, edit, hapus produk
- Field: nama, deskripsi, harga jual, harga modal, stok, kategori
- Catatan/notes untuk produk (tampil di bot setelah pembelian)
- Toggle aktif/nonaktif produk

### Varian Produk
- Produk bisa punya parent (produk induk) sebagai varian
- Contoh: "Netflix 1 Bulan", "Netflix 3 Bulan" sebagai varian dari "Netflix"
- Catatan varian dengan fallback ke catatan parent
- Tampilan terjual per varian di bot

### Kode Akun (Stok Digital)
- Upload kode akun via file (drag-drop) atau input manual
- Format: satu kode per baris
- Stok otomatis terupdate saat kode terpakai
- Riwayat penggunaan kode (dipakai oleh siapa, kapan)

### Diskon Massal (Bulk Pricing)
- Set minimum qty dan persen diskon di level produk
- Varian mewarisi bulk pricing dari produk induk
- Indikator diskon tampil di katalog bot

### Kategori
- CRUD kategori dengan emoji
- Produk difilter berdasarkan kategori di katalog bot

## 6. Manajemen Order

- Tabel semua transaksi dengan filter status
- Status order: Pending → Paid → Completed / Failed / Cancelled
- Detail order: produk, qty, harga, metode bayar, kode akun yang dikirim
- **Override status manual** oleh admin (untuk troubleshooting)
- Kirim ulang kode akun ke user
- Riwayat order bisa dicari/difilter berdasarkan tanggal, status, user

- Riwayat semua deposit dengan status
- Detail: user, jumlah, metode, waktu, status
- Konfirmasi deposit manual oleh admin
- Notifikasi otomatis ke user saat deposit dikonfirmasi

## 9. Pengaturan Sistem

### Bot Manager
- Start / Stop / Restart bot dari UI
- Tampilan log bot secara live

### Payment Gateway
- Toggle aktif/nonaktif per gateway (PakKasir, GoPay, OrderKuota)
- Input konfigurasi API key, slug, secret per gateway
- Pilih gateway default untuk tampilan di bot

### Tampilan Toko
- Ubah nama toko
- Upload/ganti banner gambar toko (tampil di setiap layar bot)

### Backup
- Backup otomatis terjadwal (interval dapat dikonfigurasi)
- Dua mode: Full backup atau Database saja
- File backup dikirim ke Telegram log chat
- Backup manual on-demand dari panel

## 10. Keamanan

- Autentikasi admin menggunakan header `x-admin-key`
- Password admin di-hash dengan **bcrypt**
- Proteksi race condition pada pembelian saldo via **database transaction atomik**
- Conditional balance update (`WHERE balance >= amount`) mencegah saldo minus
- Double-click protection dengan debounce 1.5 detik pada tombol bot
- CAS pattern (`WHERE status = 'pending'`) mencegah double-konfirmasi order
- Aksi sensitif (deposit, profile) hanya di Private DM, tidak di grup

## 11. Arsitektur Teknis

| Komponen | Teknologi |
|---|---|
| Bot Backend | Node.js + TypeScript + `node-telegram-bot-api` |
| Admin Panel | React + Vite + Tailwind CSS + Shadcn UI |
| Database | PostgreSQL + Drizzle ORM |
| Build System | pnpm monorepo (workspace) |
| Server | Express.js, webhook mode |
| Logging | Pino (JSON structured logs) |
| File Upload | Multer (drag-drop zona di admin panel) |

### Monorepo Structure
```
/
├── artifacts/
│   ├── api-server/       # Bot + REST API backend
│   └── admin-panel/      # React admin dashboard
└── lib/
    └── db/               # Shared database schema (Drizzle ORM)
```

### Database Tables
- `users` — data pengguna dan saldo
- `products` — produk dan varian
- `product_codes` — stok kode akun digital
- `categories` — kategori produk
- `orders` — riwayat transaksi pembelian
- `deposits` — riwayat top-up saldo
- `ratings` — penilaian produk oleh user
- `bot_stats` — statistik penggunaan bot
